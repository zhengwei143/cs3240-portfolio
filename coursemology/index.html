Coursemology

Introduction

Coursmology is an online platform for teachers or professors to create online courses (mainly targetted at Programming) with all the functionalities that they would need to run the course. Features include:


  • Online assessments for Continual Assessment
  • Announcements
  • Forum for discussion
  • Inline annotations for comments on code
  • Workbin for uploading of materials
  • Lesson Plan (Timeline)
  • Surveys
  • Videos

Although this is just a subset of the features available on the application, it should provide quite a clear picture of what it aims to do: to provide a wholesome experience for educators to run their courses through the school term. In addition, Coursemology adds a spin to things by gamifying the entire course, where students are encouraged to do more practices and score higher in order to gain more "experience" to level up. This encourages students adds the element of fun to the learning process for students, and makes learning a little less boring.


Task

I was tasked with developing the backend and building the frontend User Interface for the creation of questions and test cases for C/C++ and Java questions. While there was already an existing User Interface that was built for Python questions, this task turned out to be not as simple as I initially thought.


The images above was the (then) existing User Interface used to create the questions and test cases. As seen, the interface was a mix of both code editors as well as proper elements that pieced together a single question. This was due to the nature of the problem, the implementation required both a mix of flexibility (in being able to create all sorts of questions) as well as an intuitive graphical design.


Although it might have been nice to do away with the code editors, it turned out to be a must-have feature as without them, it would be difficult for the user to create any questions to begin with. The code editors sacrificed a bit of the intuitiveness of the user experience for flexibility and functionality.


Designs & Thoughts

Due to differences in implementation in the backend (for C/C++/Java and the already existing implementation - Python), I found that I was at the same crossroads with a choice to make - Inituitiveness against Functionality. Of course, I choice the latter. Given that the setup of questions was mainly a one-time thing at the start of the semester, the effect of the lack of intuitiveness was far outweighed by the flexibility and functionality that came with the less intuitive design.

There were, however, places were I could further improve on the design of the User Interface. Previously, variables were all declared in the code editors on the left, which meant that users would have to constantly scroll up and down between the above 2 screens (the left screen is above the right screen on the application) to refer to the variables when creating test cases.


I added additional code editors under each test case, so that code related specifically to each test case would be situated next to the (graphical) test case itself. This way, it became much more convenient and intuitive for users when creating large numbers of test cases for the question.
